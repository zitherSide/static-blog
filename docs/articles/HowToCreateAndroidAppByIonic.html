単語帳アプリ。個人的に欲しかったし、機能は単純だったので作ってみた。

作ったモチベーション

単語の日本語があんまりピンとこなくてなかなか覚えられない単語があるときにGoogle画像検索をつかって「あぁこんなイメージか」って確認するんだけど、ある日

「これわざわざ自分で画像検索するの面倒じゃね？」

と思って自動的にGoogle画像検索のページに飛ばしてくれる単語帳を作った。

例えばlicoriceって単語知らなくて調べたら「リコリス菓子」と出てきて「は？その日本語もわからないんだが？」ってなった。

ちなみにリコリスはこんなお菓子らしい

https://www.google.com/search?q=licorice&sxsrf=ALeKk03yVxWS_GRWVhPqaB-y7hAwIp31ug:1623497514091&source=lnms&tbm=isch&sa=X&ved=2ahUKEwjG2vvE_5HxAhVGUd4KHQfxCVAQ_AUoAnoECAEQBA&biw=1815&bih=1009

日本人の口にはあまりあわないっぽい。

というわけで主な機能は以下の通り

主な機能

  1. 英単語は自分で入力

  2. 答えは入力せず、Googleの画像検索、普通の検索、動画検索に飛ばす

  3. 不正解, 正解, 簡単、を選択できて勝手に次回学習日をスケジューリングしてくれる

  4. 単語一覧を表示できて次回学習日や学習の間隔を手動でも編集できる

vuexの準備

さっそく実際のコードをみてみる。

store/index.ts
import { InjectionKey } from 'vue' (1)
import { createStore, useStore as baseUseStore, Store, MutationTree, ActionTree } from 'vuex'

export interface Word {
    word: string;
    expiration: number;
    lastTerm: number;
}

export interface State {
    words: Word[];
}

export const key: InjectionKey<Store<State>> = Symbol('') (2)
const state: State = {
    words: []
}

export function useStore(){
    return baseUseStore(key) (3)
}

export const enum MUTATIONS { (4)
    ADD_WORD = 'ADD_WORD',
    REMOVE_WORD = 'REMOVE_WORD',
    UPDATE_WORD = 'UPDATE_WORD',
    SHIFT_ROTATE = 'SHITF_ROTATE'
}

const mutations: MutationTree<State> = {
    [MUTATIONS.ADD_WORD](state, word: Word){
        state.words.push(word)
    },
    [MUTATIONS.REMOVE_WORD](state, word: string){
        state.words = state.words.filter( elem => elem.word !== word)
    },
    [MUTATIONS.UPDATE_WORD](state, word: Word){
        const index = state.words.findIndex( elem => elem.word === word.word)
        if(index == -1){
            state.words.push(word)
        }else{
            state.words[index] = word
        }
    },
    [MUTATIONS.SHIFT_ROTATE](state){
        const shifted = state.words.shift()
        if(shifted)
            state.words.push(shifted)
    }
}

export const store = createStore<State>({state, mutations}) (5)

  2. Symbolを使ってKeyを作る

  3. 使いやすいように外部にはkeyバインドした状態で公開する

  4. タイポとかしないようにmutations用の文字列を定義しておく。別に必須ではない

  5. stateとmutationsを渡してStoreを作ってexport

@capacitor/storageの利用

storeの準備ができたので、データを永続化するために@capacitor/storageも使う。

vuex storeにstorageに保存とstorageから読み出しするactionsを追加する。 FETCH_WORDSでストレージから読み出し、SAVE_WORDSでストレージに保存する。

store/index.ts
export const enum ACTIONS {
    FETCH_WORDS = 'FETCH_WORDS',
    SAVE_WORDS = 'SAVE_WORDS'
}

const actions: ActionTree<State, any> = {
    [ACTIONS.FETCH_WORDS](store){
        Storage.keys().then( keysRes => { (1)
            keysRes.keys.forEach( async key => {
                const valStr = await Storage.get({key: key}) (2)
                const val = JSON.parse(valStr.value || '')
                store.commit(MUTATIONS.ADD_WORD, {
                    word: key,
                    expiration: FloorDate(val.expiration),
                    lastTerm: val.lastTerm as number
                })
            })
        })
    },
    [ACTIONS.SAVE_WORDS](store){
        store.state.words.forEach( word => {
            const key = word.word
            const valObj = {
                expiration: FloorDate(word.expiration),
                lastTerm: word.lastTerm
            }
            const val = JSON.stringify(valObj)

            Storage.set({ (3)
                key: key,
                value: val
            })
        })

    }

}

  2. Storage.getでKeyを指定することでストレージからデータを読み出せる

  3. Storage.setでオブジェクトをストレージに保存できる

Google Playに公開する

Google Playへのアプリ登録はGoogle Play Console https://play.google.com/console/signup から行う。

アプリの登録やアップグレードはGoogle Play Consoleで リリース > 製品版 > 新しいリリースを作成　でおこなう。

Google Playに登録するためにionic capacitorでビルドしたものから変更しなきゃいけない点はこの3つ。

  1. パッケージ名を一意なものに変更

  2. バージョンを適切に付与

  3. アイコンを設定する

それぞれ直接androidのファイルを編集する。(と思ってるけどcapacitorのcapacitor.config.tsから設定できるのかな？)

パッケージ名

Google Playで公開するためにはパッケージ名は一意に決まらなきゃいけないので設定する。

ionic capacitorでビルドするでデフォルトのパッケージ名は

 io.ionic.starter.[App名]

とかになってると思う。

パッケージ名は一意な文字列ならなんでもいいっぽいけど、今回は

zitherSide.github.io.FlashImageCard

にする。(逆順にしてio.github.zitherSide.FlashImageCardのが良かったかも？)

置き換えはプロジェクトの全置換でゴリ押しした。(もっと他にやり方がある気がする…​)

ちなみに自分でandroidプロジェクトのファイルを書き換えた部分は下記コマンドでリビルドしても、そのままちゃんと変更した分が残ってくれた。

> ionic build
> ionic cap copy

バージョン

Google Playで公開したあとにバージョンアップをするには適切にバージョン番号をインクリメントして上げる必要がある。

バージョン番号の設定はandroid/app/build.gradleを編集する。

android/app/build.gradle
android {
    ...
    defaultConfig {
        ...

  2. ストアで表示されるバージョン。なんでも良い。

Ionicで初期化処理

基本的に公式ページの通りに実装する。今回はバナー表示広告にした。

常に画面下に表示するようにしたいのでvueのmoutedフックでバナー表示関数を呼ぶことにした。(個人的に広告の場所が変わるせいで間違ってタップしちゃうのが一番嫌なので常時固定位置表示にした)

広告の初期化はmain.tsで行って、表示関数の定義はApp.vueで行って各ページのmountedで表示した。

表示関数もmain.tsからexportしても良かったかもしれないけど、あんまりmain.tsをimportしてるファイルを見たことないのでやめておいた。

まずはmain.tsでAdmobを初期化する。

main.ts
import { AdMob } from '@capacitor-community/admob';

export async function initialize(): Promise<void> {
  AdMob.initialize({
    requestTrackingAuthorization: true,
    testingDevices: ['2077ef9a63d2b398840261c8221a0c9b'], (1)
    initializeForTesting: true,
  });
}

  1. テストデバイスのIDはサンプルからのそのままコピペしてきたけど、普通に自分のデバイスでも表示された

次にApp.vueにバナー表示関数を宣言してexportする。基本的にサンプルコードコピペ。

import { AdMob, BannerAdOptions, BannerAdSize, BannerAdPosition, BannerAdPluginEvents, AdMobBannerSize } from '@capacitor-community/admob';

export async function banner(): Promise<void> {
    AdMob.addListener(BannerAdPluginEvents.Loaded, () => {
      // Subscribe Banner Event Listener

    AdMob.addListener(BannerAdPluginEvents.SizeChanged, (size: AdMobBannerSize) => {
      // Subscribe Change Banner Size
    });

    const options: BannerAdOptions = {
      adId: 'YOUR ADID', (1)
      adSize: BannerAdSize.BANNER,
      position: BannerAdPosition.BOTTOM_CENTER,
      margin: 0,
      isTesting: true,
      npa: true
    };
    AdMob.showBanner(options).then( (2)
      value => console.log('success', value),

    )
}

android用の初期化処理

ionicの実装が終わっても、admob自体のandroid対応とか、capacitorプラグインの初期化などがandroid側で必要。

  1. AndroidManifest.xmlにID登録

  2. build.gradleに依存関係を追加

  3. MainActivityでPlugin初期化

AndroidManifest.xmlにIDを登録

admobで発行されたAppIDをリソースに登録する。AdIDではないので注意。

AndroidManifest.xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="zitherSide.github.io.FlashImageCard">

    <application
        ...>

        <meta-data
            android:name="com.google.android.gms.ads.APPLICATION_ID"
            android:value="AppID"/>
        ...
    </application>

依存関係の追加

プロジェクトのbuild.gradleに

build.gradle
allprojects {
    repositories {
        google()
    }
}

アプリのbuild.gradleに

build.gradle
dependencies {
    implementation 'com.google.android.gms:play-services-ads:20.1.0'
}

を追加する。

実行しようとしてもエラーになるときはメニューからBuild/Make Projectすると直る。

MainActivityでPlugin初期化

MainActivityのonCreateでadmobプラグインの登録を行う。

MainActivity
import android.os.Bundle;

import com.getcapacitor.BridgeActivity;
import com.google.android.gms.ads.MobileAds;
import com.google.android.gms.ads.initialization.InitializationStatus;
import com.google.android.gms.ads.initialization.OnInitializationCompleteListener;

public class MainActivity extends BridgeActivity {
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        registerPlugin(com.getcapacitor.community.admob.AdMob.class);
    }
}

これでビルドすれば無事広告がついてGoogle Playに公開できる。
