1. 全体の流れ

1.1. 完成品

出来上がったGitHub Pages上のサイトはこちら https://zitherside.github.io/static-blog/

Nuxt.jsを使って静的Webサイトを生成して、GitHub Pagesにデプロイする。

npm run generate

コマンドを使えば、Node.jsのない環境でも動くWebアプリを生成してくれるので、これらなGitHub Pagesにおいても見れる。

以下の順で作業する。

  1. 静的サイトのビルド

  2. GitHub Pagesへのデプロイ

  3. Vuexでデータ定義

  4. Jestでテスト作成

  5. Ascii Docで作ったページをNuxtに読み込み

  6. G-Driveへのファイルリンク

  7. 一覧ページの作成と検索

2. Nuxt Generateで静的サイトをビルド

まずはとりあえず静的サイトをビルドできるようにする。

npx create-nuxt-app
npm run generate

ちゃんと生成されることを確認したら、実際の開発中は変更点をホットリロードしてほしいので、いつもどおりのdevコマンドでサーバーを立てる。

npm run dev

3. GitHub Pagesにデプロイ

ビルドできることを確認したら、中身を開発する前にまずGitHub Pagesにちゃんとデプロイできることを確認する。

3.1. GitHub側の設定

GitHub Pagesで公開するブランチと対象のフォルダをGitHubから設定する。今回は

  • gh-pages ブランチ

  • /docs フォルダ

を設定。 github → settings → pages で設定する。

Caution
 github pagesはgithub actionsでデプロイが行われるので、設定を反映したページにするには、何かしらpushする必要がある。

3.2. Nuxt側の設定

出力フォルダ設定

docsフォルダを公開するので、generateコマンドの出力先をdocsに設定する。 nuxt.config.jsonをこんな感じで編集する。

nuxt.config.json
 generate: {
    dir: 'docs'
  },
ベースディレクトリの設定

デプロイしたときにリンク先が正しくなるようにベースディレクトリをgithub pagesに合わせる必要がある。 同じくnuxt.config.jsonに設定する。 github pagesのベースURLはリポジトリ名になるので ` /[repository name]/ ` を設定する。今回はこんな感じ。

nuxt.config.json
  router: {
    base: '/static-blog/'
  },

3.3. GitHub Actionsの設定

GitHubで設定したので、gh-pagesにpushされれば, docフォルダ以下がgithub pagesに自動的にデプロイされる。 だが、masterにpushされたソースをもとに nuxt generate して、github-pagesブランチにpushするワークフローは自分で作る必要がある。

やることは4つ * ソースのcheckout * checkout@v2を使用 * node.js環境のセットアップ * setup-node@v2を使用 * nuxt generate * gh-pagesへpush * actions-gh-pages@v3を使用

workflowファイルはこんな感じ。

.github/workflow/pages-build-deployment.yaml
name: build and deploy

on:
  push:
    branches: [ main, master ]

jobs:
  build-and-deploy:

    runs-on: ubuntu-latest

    strategy:
      matrix:
        node-version: [12.x, 14.x, 16.x]
        # See supported Node.js release schedule at https://nodejs.org/en/about/releases/

    steps:
    - name: checkout
      uses: actions/checkout@v2
    - name: Use Node.js ${{ matrix.node-version }}
      uses: actions/setup-node@v2
      with:
        node-version: ${{ matrix.node-version }}
        cache: 'npm'
    - name: build
      run: |
        npm install
        npm run generate
    - name: deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
        branch: gh-pages
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./docs
        destination_dir: ./docs
Note
 サンプル通りにnode.jsのバージョンを12,14,16とmatrixで指定したが、16だけで良かったかもしれない\

3.4. 発生したエラー

3.4.1. フォルダの指定ミス

 Conversion error: Jekyll::Converters::Scss encountered an error while converting 'assets/css/style.scss': No such file or directory @ dir_chdir - /github/workspace/docs

actions-gh-pagesのdestination_dirの指定を ./doc にしたら直った。scssがない、と言っているが、原因は/github/workspace/docsフォルダに結果が置かれていないこと。

3.4.2. 変更履歴のコンフリクト

error: failed to push some refs to 'https://github.com/zitherSide/static-blog.git'
  hint: Updates were rejected because the remote contains work that you do
  hint: not have locally. This is usually caused by another repository pushing
  hint: to the same ref. You may want to first integrate the remote changes
  hint: (e.g., 'git pull ...') before pushing again.
  hint: See the 'Note about fast-forwards' in 'git push --help' for details.
  Error: Action failed with "The process '/usr/bin/git' failed with exit code 1"

リモート側にローカルにない変更があるせいで失敗してる。 自分でpushする場合はforce-with-leaseなどのオプションで強制的にpushすればよいが、actionc-gh-pages を使っているので、手動で一旦リモート側のブランチを消して、 コンフリクトしないようにしたら直った。

4. Vuexでデータ定義

個人的には最初にデータ定義を決めるのが好きなので、まずinterface定義をする。 最初にDBのテーブル定義をやるような感覚。

Vuex でデータ管理するので /store/index.ts にinterfaceを定義。

/store/index.ts
export interface ArticleData{
    title: string
    path: string
    isRawHtml: boolean
    thumbnail: string
    tags: Tag[]
    category: Category
    createdAt: Date
}

5. Jestでテスト作成

データ構造の定義ができたら、Jestを使ってテストを作る。

5.1. Typescrip型情報をinstall

Typescriptで型定義したので、typescript用の型定義情報をインストールする。 開発環境だけで使ってデプロイ先では必要ないので --save-devでインストール

npm i --save-dev @types/jest

tsconfig.json にも設定を追加する。 デフォルトで node_modules/@types にある全ての型を読み込んでくれるが、nuxtが作ってくれるtsconfig.jsonは読み込むフォルダを指定してあるので、 それにならって今回インストールしたJestの型情報も追加する。

tsconfig.json
{
  "compilerOptions": {
    "types": [
      "@nuxt/types",
      "@types/node",
      "@types/jest"
    ]
  }
}

5.2. Testの作成

まずはとりあえず動くテストを作履帯ので、testフォルダの中に article.spec.ts を作成する。 .*\.spec\.(js|ts)$となっているファイルをテストケースとして認識してくれる。

Describeでテストスイートを定義して、itでテストケースを定義する感じ。 いわゆるビヘイビア駆動開発で使いやすいようになっている。動作(Behavior)を記述する(Describe it)、という自然言語として読めなくもないような形でテストコードを記述する。

test/article.spec.ts
import { ArticleData } from '~/store/index'

describe('ArticleIFTest', () => {
  it('set_and_get', () => {
    const art: ArticleData = {
      title: 'title',
      path: 'path',
      thumbnail: '/uri/thumbnail',
      tags: ['tag1', 'tag2'],
      category: 'category',
      createdAt: new Date()
    }

    expect(art.title).toBe('title')
    expect(art.tags.length).toBe(2)
  })
})

Describeにもitにも関数を渡して記述する。itの中にはexpectを含んだテスト関数自身を記述し、describeの中にはit関数やbeforeEach関数などを呼んでテストの登録をする関数を記述する。

できたら

npm run test

でテストを実行する。

5.3. VueComponent、Veutiifyのテストを書く

Jestのテストを作る事ができたので、Vuetifyを使ったVueのComponentのテストを書く。 Vueのテストを書くためにはvue/test-utilsを使ってテストで使うためのVueオブジェクトを作成してあげる必要がある。

今回テストするページでは下記の機能を使うための準備が必要。

  • Vuetify

  • Vuex

  • VueRouter

VuexとVueRouterはテスト定義の中でモックを宣言して使う。 まずはvue/test-utilsのcreateLocalVueヘルパーとshallow-mountを使ってテスト用のVueオブジェクトを作成する。

このビヘイビア内の全てのテストケースで使うので、Describeのスコープでテスト用Vueオブジェクトとしてwrapperを宣言する。 BeforeEach関数の中でwrapperにモックとして作ったオブジェクトをマウントして各テストで使えるようにする。

マウントのUtil関数は mount と shallow-mount があるが、shallow-mount の方はvueテンプレートファイルをレンダリングした結果がスタブになる。単体テストとして、このファイルにあるscriptのロジックを確かめたいだけなので shallow-mount を利用した。

test/default.spec.js
import { createLocalVue, shallowMount } from '@vue/test-utils'
import VueRouter from 'vue-router'
import Vuex from 'vuex'
import Vuetify from 'vuetify'
import Vue from 'vue'


describe('defaultLayout', () => {
  let wrapper
  let articlesStoreMock
  let store
  let vuetify
  const routes = [{ path: '/' }]
  const router = new VueRouter({
    routes,
    base: '/base/'
  })

  beforeEach(() => {
    const localVue = createLocalVue()
    vuetify = new Vuetify()
    localVue.use(VueRouter)
    localVue.use(Vuex)
    localVue.use(Vuetify)
    articlesStoreMock = {
      namespaced: true,
      getters: {
        get: () => [
          //testでstoreからの入力として使いたいデータを定義
        ]
      }
    }
    store = new Vuex.Store({
      modules: {
        articles: articlesStoreMock
      }
    })
    wrapper = shallowMount(Layout, {
      localVue,
      router,
      store,
      vuetify
    })
  })
})

5.3.1. 発生したエラー

[Vue warn]: Unknown custom element: <v-container> - did you register the component correctly?

VuetifyをUseしていないことが原因。 localVue.Use(Veutify) して、shallowMount でvuetifyオブジェクトをマウントしてあげたら直った。

5.4. VSCodeでステップ実行する

テストが書ける様になったので、VSCodeでステップ実行してデバッグできるようにする。 Jestの公式ページ[https://jestjs.io/ja/docs/troubleshooting]の指示に従って、launch.jsonにコンフィグを追加する。

.vscode.launch.json
{
    "configurations": [
        {
            "name": "Debug Jest Tests",
            "type": "node",
            "request": "launch",
            "runtimeArgs": [
              "--inspect-brk",
              "${workspaceRoot}/node_modules/jest/bin/jest.js",
              "--runInBand"
            ],
            "console": "integratedTerminal",
            "internalConsoleOptions": "neverOpen",
            "port": 9229
          }
    ]
}

やってることは type にnodeが設定してある通り、普通にNodeの実行をしているだけ。

runtimeArgs でNodeに対して --inspect-brk でブレイクポイントを設定。 実行するファイルはjest.jsを指定して、jestのオプションとして -runInBand でテストがシリアルに実行されるようにしている。

5.5. Coverageの計測対象設定

せっかくなのでカバレッジも計測できるようにする。 jest.config.js にJestの設定を記述できるので、collectCoverageFrom プロパティを設定する。

jest.config.js
module.exports = {
  collectCoverageFrom: [
    '<rootDir>/components/**/*.vue',
    '<rootDir>/pages/**/*.vue',
    '<rootDir>/layouts/**/*.vue'
  ]
}

6. Ascii DocによるページをNuxtで表示する

文章の多いページを頑張ってNuxtの中で書くのは大変なので、AsciiDocで書いた結果をそのまま表示したい。 レイアウトを含めてそのまま使いたいので、タグなどをエスケープせずにhtmlの内容をそのまま展開してくれる v-html を使う。

static/articles フォルダにAsciiDocから出よくしたhtmlを配置して、それを読み込んで表示してくれるページを作る。

  1. static/articlesにファイルを配置

  2. pages/articles/_article.vue でファイル情報をクエリで受け取って動的に表示

  3. Vuexで記事のタイトル、タグ、ファイルパスなどを管理

というわけでこんなvueテンプレートファイルを作る。一応 v-ifを使って読み込み中の表示も作った。

pages/articles/_article.vue
<template>
  <div>
    <div v-if="!loaded">
      <v-progress-circular v-if="!loaded" indeterminate color="primary" />
      <span>loading...</span>
    </div>
    <div v-else v-html="content" />
  </div>
</template>

クエリに従ってhtmlファイルを読み込むスクリプトはこんな感じ。渡されたクエリは this.$route.quetyでアクセスできる。

pages/articles/_article.vue
export default {
  data: () => {
    return {
      content: '',
      loaded: false
    }
  },
  mounted () {
    const validPattern = /\/static-blog\/articles\/.*\.html$/
    if (!validPattern.test(this.$route.query.path)) {
      console.log('invalid URL')
      return
    }

    fetch(this.$route.query.path)
      .then(data => data.text())
      .then((text) => {
        this.content = text
        this.loaded = true
      })
      .catch((err) => { console.log(err) })
  }
}

ファイル名を _article.vue とアンダースコアで始めることで、パスをパラメータとして受け取るようにしてある。 これをしないとクエリを変えてもNuxtが同じページだと判断してキャッシュから以前のパラメータで読み込んだ内容をそのまま表示してしまい、 内容が切り替わらないことがあった。

Caution
 v-htmlはタグなどをエスケープせずにそのまま展開するのでXSSの危険があるので、普通のサーバーサイドレンダリングのサイトでは使わないようにしたほうが良い。今回は静的サイト生成なので問題ないが、サーバーサイドレンダリングだと不正なスクリプトの書かれたページのURLをクエリに渡されて悪意のあるスクリプトが実行できてしまう。

クエリを渡すのは、 to に path プロパティと query プロパティを持つオブジェクトをバインドすれば良い。

<v-list-item :to="item.to">
item: {
  to: {
    path: `/articles/${category}${index}`,
    query: {
      path: `${basePath}${x.path}`
    }
  }
}

7. G-Driveのファイルにリンクする

Unreal Engineでつくったゲームを何処かで公開しようと想ったけど、何も考えずに適用にアセットを入れて作った結果容量が5GB以上になって、無料で公開できるページがあまり見つからなかったのでG-Driveを公開設定にして利用することにした。

ファイルの共有設定で「リンクを知っている人全員」にすると誰でもファイルがダウンロードできるようになる。 Nuxtのページからをそこへのリンクを貼る。

https://drive.google.com/file/d/1_7YZibLcYHlPam_pBbRBN-dDKUhFM2oa/view?usp=sharing

同じアプリ内のページであれば to にバインドすることでリンクできるが、外部サイトなので href にバインドする必要がある。

to は不必要なリロードを行わない、など同一アプリ内でのリンクのためのプロパティなので使えない。

<v-card-actions>
        <v-btn class="ml-auto mt-auto white--text" href="https://drive.google.com/file/d/1_7YZibLcYHlPam_pBbRBN-dDKUhFM2oa/view?usp=sharing" depressed color="black">
          Download
          <v-icon color="red darken-2">
            mdi-download
          </v-icon>
        </v-btn>
      </v-card-actions>

8. 記事の一覧をカテゴリとタグで検索できるようにする

記事の一覧をテーブル表示して、カテゴリやタグで検索できるようにする。 完成品ページはこんな感じ。

https://zitherside.github.io/static-blog/list

選んだTagとCategoryでfilterするcomputedプロパティを作成し、v-data-table の items にバインドすれば、カテゴリとタグで検索できるようになる。

<v-data-table :headers="headers" :items="articles" :search="search" @click:row="jump">
      <template #[`item.tags`]="{ item }">
        <v-chip
          v-for="(tag, i) in item.tags"
          :key="i"
          :color="$store.getters['attributes/tagColors'][tag]"
          text-color="white"
        >
          {{ tag }}
        </v-chip>
      </template>
    </v-data-table>
  computed: {
    articles () {
      return getters.get(state())
        .filter(item => this.searchCategories.includes(item.category))
        .filter(item => item.tags.some(tag => this.searchTags.includes(tag)))
    }
  }
